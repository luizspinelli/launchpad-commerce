## 📋 What Is BMAD-METHOD?

BMAD (Breakthrough Method for Agile AI-Driven Development) is a framework that guides project development from discovery → architecture → implementation using AI-assisted workflows and domain experts.

**For this project:**
- Each project has **isolated BMAD instance** (not shared)
- This folder (`.bmad/`) contains all planning & decision artifacts
- BMAD helps you answer the "what" & "why" before coding the "how"

## 🎯 BMAD Workflow Phases

### Phase 1: Discovery (START HERE)

**Goal:** Understand project requirements, scope, risks

**Command in IDE:**
```
@bmad-help

# Or ask directly:
@bmad-pm: I want to build a landing page + e-commerce with Stripe. 
           What questions do I need to answer first?
```

**Outputs go to:** `.bmad/discovery/`

**Questions BMAD will guide you through:**
- [ ] Who is your user?
- [ ] What problem does it solve?
- [ ] What's MVP vs. nice-to-have?
- [ ] What are the risks?
- [ ] What's the success metric?

**Timeline:** 1-2 hours

### Phase 2: Architecture (NEXT)

**Goal:** Design system, make tech decisions, document trade-offs

**Command in IDE:**
```
@bmad-architect: Based on discovery, design the system architecture.
                 We're using Next.js 14, Stripe, Resend.
                 Generate system design + component architecture.
```

**Outputs go to:** `.bmad/architecture/`

**Deliverables:**
- System design diagrams
- Component architecture
- API contracts
- Database schema
- Tech stack rationale

**Timeline:** 1-2 hours

### Phase 3: Implementation (THEN)

**Goal:** Break down into tasks, plan sprints, testing strategy

**Command in IDE:**
```
@bmad-developer: Create implementation roadmap.
                 Timeline: 1-2 weeks. Team: 1 dev (me).
                 Generate dev tasks, testing strategy, scaffold.
```

**Outputs go to:** `.bmad/implementation/`

**Deliverables:**
- Week-by-week task breakdown
- Sprint tasks (1-day tasks)
- Testing strategy
- Code scaffolds (templates)

**Timeline:** 1 hour

## 🎤 Talking to BMAD Agents

BMAD has specialized agents (personas). You can invoke them:

### Project Manager (@bmad-pm)
```
@bmad-pm: What are the key success metrics for LaunchPad?
          How do we measure if it's working?
```

### Architect (@bmad-architect)
```
@bmad-architect: Design the component hierarchy for the landing page.
                 I want sections for hero, features, testimonials, CTA.
```

### Developer (@bmad-developer)
```
@bmad-developer: Break down the Stripe integration into tasks.
                 How many days for each? What's the dependency order?
```

### QA/Test Architect (@bmad-test-architect)
```
@bmad-test-architect: Create testing strategy for checkout flow.
```

```
@bmad-ux-designer: Design the landing page wireframes.
                   Key sections: hero, features, testimonials, CTA.
```

## 📝 Running a Full Workflow

### Day 1: Discovery
```bash
# Morning: Install BMAD
npx bmad-method install

# Afternoon: Ask PM questions
@bmad-pm: Guide me through discovery for LaunchPad.
          What do I need to know about my users, market, competitors?

# Evening: Review & commit
git add .bmad/discovery/
git commit -m "Discovery: User personas, requirements, MVP scope"
```

### Day 2: Architecture
```bash
# Morning: Architect designs system
@bmad-architect: Based on our discovery, design the full system.
                 Components, API, database, deployment.

# Afternoon: Create ADRs (decision records)
# -> Already have 3 ADRs (Next.js, Stripe, Vercel)

# Review decisions
git add .bmad/architecture/ .bmad/decisions/
git commit -m "Architecture: System design, tech stack decisions, ADRs"
```

### Day 3: Implementation Planning
```bash
# Morning: Developer breaks down tasks
@bmad-developer: Create detailed implementation roadmap.
                 Week 1: Setup, landing page, product pages
                 Week 2: Stripe, email, thank-you flow

# Generate code scaffold
@bmad-developer: Generate Next.js project scaffold.
                 Include: app/page.tsx, app/products/, app/api/stripe/

# Afternoon: Create tests strategy
@bmad-test-architect: Testing plan for checkout flow
                      Unit tests, integration tests, E2E tests

git add .bmad/implementation/
git commit -m "Implementation: Dev roadmap, testing strategy, code scaffold"
```

### Day 4-14: Actual Development
```bash
# Now you have clear roadmap
# Follow .bmad/implementation/dev-roadmap.md

npm install  # Install dependencies
npm run dev  # Start dev server

# Build pages, API routes, etc as planned
# Reference BMAD outputs when unsure
```

## 🚨 Tips & Gotchas

### ✅ DO

- **Commit after each phase** (`git add .bmad/ && git commit`)
- **Reference ADRs** when making decisions (already have 3)
- **Use BMAD outputs** as source of truth
- **Ask for clarification** if BMAD response is unclear
- **Export diagrams** for stakeholder presentations

### ❌ DON'T

- **Skip discovery** (tempting to jump to code)
- **Ignore ADRs** (decisions need context)
- **Lose BMAD outputs** (git should have everything)
- **Treat BMAD as gospel** (it guides, you decide)
- **Forget to update** `.bmad/` as things change

## ❓ FAQ

**Q: Do I need to run all BMAD workflows?**  
A: Discovery + Architecture are essential. Implementation planning is helpful but can be done manually if needed.

**Q: Can I skip BMAD and just code?**  
A: You can, but you'll waste time on decisions that BMAD helps clarify upfront.

**Q: Will BMAD write code for me?**  
A: BMAD generates scaffolds, templates, and architecture. You write the actual production code.

**Q: How long does BMAD take?**  
A: ~3-4 hours for discovery + architecture. Saves ~20+ hours of unclear planning.

**Q: Can I use BMAD for all 3 projects?**  
A: Yes! Each project has its own `.bmad/` folder (isolated instances).
